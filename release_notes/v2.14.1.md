# v2.14.1

**Release Date:** April 6, 2026

## 🐛 Bug Fixes

### Manual Override Repositioning (#132)
Fixed covers repositioning outside the configured active time window when a manual override expires. Previously, when a user manually moved a cover outside active hours and the override timer elapsed, the integration would attempt to send a new position even though the active time window was closed. Covers now respect the time window boundary and only reposition when re-entering an active window.

### Quick Setup Configuration Crash (#133)
Fixed `None` values being stored in configuration when skipping optional setup fields. When users progressed through the quick setup without filling certain optional fields, those fields could be saved as `None` instead of being omitted, causing configuration load errors on subsequent restarts. Optional fields are now properly skipped if not provided.

### Climate Mode Temperature/Presence Not Reading (#134)
Fixed climate mode never activating because temperature and presence entities were not being passed to the climate provider. Introduced in v2.12.0, the climate reading code was missing entity IDs in the provider call, causing `inside_temperature`, `outside_temperature`, and `is_presence` values to always be `None`. Climate mode strategies (summer/winter) never activated — covers silently degraded to solar-only tracking. Fixed by ensuring all configured climate entities are properly passed to the provider.
