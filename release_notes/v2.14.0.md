# v2.14.0

## 🐛 Bug Fix: Constant Repositioning at 0%/100% (#127)

Covers configured with a minimum of 0% and maximum of 100% were receiving repeated positioning commands every few minutes when already fully open or closed. This unnecessarily cycled actuators and generated log noise.

**Root cause:** The special-position delta bypass (designed to allow smooth transitions *to* key positions like 0% and 100%) did not check whether the cover was *already* at the target. A cover sitting at 0% with a target of 0% would bypass the delta check and only be gated by the time threshold — so it was re-commanded every `time_threshold` minutes indefinitely.

**Fix:** A same-position short-circuit now runs first: if the current position equals the target, no command is sent regardless of special-position status. Transitions to/from 0%/100% continue to work exactly as before.

## ✨ Feature: Configurable Priority per Custom Position Slot

Each of the 4 custom position slots now has its own independent pipeline priority (1–99). Previously all slots shared a fixed priority of 77 (between Manual Override at 80 and Motion Timeout at 75).

**Why this matters:** You can now place different custom positions at different points in the decision chain to match your automation logic:

| Priority range | Behavior |
|---|---|
| > 80 | Overrides even manual moves (acts like a soft force override) |
| 77 (default) | Original behavior — below manual, above motion timeout |
| 51–74 | Below motion timeout; only fires when room is occupied |
| < 40 | Only applies when sun tracking is inactive |

**Configuration:** In the Custom Positions settings screen, each slot now shows a Priority slider (1–99) alongside its sensor and position fields. Leave it at the default 77 if you don't need to change it.

**Backward compatible:** Existing installations without a configured priority automatically default to 77 — no reconfiguration needed.
