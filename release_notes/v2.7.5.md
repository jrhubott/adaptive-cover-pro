## ЁЯОп Adaptive Cover Pro v2.7.5

This release brings weather safety protection, occupancy-based control, improved diagnostics, and a critical bug fix for position limits.

### тЬи New Features

#### Force Override Sensors (Weather Safety)

Configure binary sensors (rain, wind, etc.) that override automatic cover control when active:

- **ANY sensor active** тЖТ all covers in the instance move to a configured safe position
- **Safe position** defaults to 0% (retracted); configurable up to 100% for security use cases
- **Manual control** still works during force override
- Use cases: rain/wind protection for awnings, security sensors for emergency access
- Configuration: `force_override_sensors` (list of entity IDs), `force_override_position` (0тАУ100%)

#### Motion-Based Automatic Control

Enable/disable sun positioning based on room occupancy using binary motion sensors:

- **ANY sensor detects motion** тЖТ covers use automatic sun-based positioning
- **All sensors show no motion** for configured timeout тЖТ covers return to default position
- Configurable timeout (30тАУ3600 seconds, default 300) prevents flapping from rapid sensor toggling
- OR logic: each integration instance can have its own set of sensors and timeout
- Configuration: `motion_sensors` (list of entity IDs), `motion_timeout` (seconds)

#### Control State Reason Sensor

New diagnostic sensor (`control_state_reason`) showing a human-readable explanation of why the cover is in its current position:

- Coordinator-level: `Force Override`, `Motion Timeout`, `Manual Override`
- Cover-level: `Direct Sun`, `Default: FOV Exit`, `Default: Elevation Limit`, `Default: Sunset Offset`, `Default: Blind Spot`, `Default`

### ЁЯРЫ Bug Fixes

- **Fixed position limit logic** (Issue #24) тАФ `min_position` and `max_position` were incorrectly applied after sunset when `enable_min_position` / `enable_max_position` was set to `True`. Covers now correctly use `sunset_position` after sunset when limits are configured as conditional.
- **Fixed "End Sun Time" sensor** тАФ `solar_times()` now respects configured elevation limits and sunset/sunrise offsets, matching actual sun tracking behavior. Previously, the reported end time could be significantly later than when the cover actually stopped tracking the sun.
