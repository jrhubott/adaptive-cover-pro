# v2.13.1 — Climate Status Sensor Fix

## 🐛 Bug Fixes

- **Climate Status sensor now working** — Fixed issue where Climate Status sensor remained in "unknown" state when climate mode was enabled. The `ClimateCoverData` was computed inside the climate handler but never propagated to diagnostics. Now correctly flows through the pipeline to display "Summer Mode", "Winter Mode", or "Intermediate".

## 🔧 Technical Improvements

- **Pipeline data propagation** — `ClimateCoverData` now included in `PipelineResult`, ensuring diagnostic builders have access to full climate state information.
- **Latent bug fix** — Fixed pipeline registry dropping the `tilt` field from handler results (preparation for Issue #33 — dual-axis venetian blinds).

## 🧪 Testing

- Added 6 new tests covering climate_data propagation through pipeline and diagnostics
- 1017 total tests passing
- Verified Climate Status sensor attributes populate correctly (active_temperature, temperature_details, climate_conditions)
