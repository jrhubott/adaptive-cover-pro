## 🎯 Control State Reason Sensor + Solar Times Accuracy Fix

### ✨ New Features

- **Control State Reason diagnostic sensor** — new P0 sensor (enabled with diagnostics) that shows a human-readable explanation for why the cover is in its current state:
  - `Direct Sun` — sun is actively tracked, cover uses calculated position
  - `Default: FOV Exit` — sun azimuth is outside the configured field of view
  - `Default: Elevation Limit` — sun elevation is outside configured min/max limits
  - `Default: Sunset Offset` — current time is within the sunset/sunrise offset window
  - `Default: Blind Spot` — sun is within a configured blind spot area
  - `Manual Override` — user manually changed the cover position
  - `Motion Timeout` — motion sensors indicate room is unoccupied
  - `Force Override` — a force override binary sensor is active

### 🐛 Bug Fixes

- **Fixed "End Sun Time" showing incorrect (too late) time** — `solar_times()` now applies the same conditions as `direct_sun_valid`:
  - Respects configured elevation limits (`min_elevation` / `max_elevation`)
  - Excludes the sunset/sunrise offset windows from the reported time range
  - Previously, "End Sun Time" could show a significantly later time than when the cover actually stopped sun tracking, causing open/close-only covers to close before the displayed end time

### 🧪 Testing

- 14 new tests for `control_state_reason` covering all reason values and priority ordering
- All 303 tests passing

Download `adaptive_cover_pro.zip` from the release assets and extract to your `custom_components` directory, then restart Home Assistant.
