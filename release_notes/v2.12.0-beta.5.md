## 📋 Override Priority Chain Visualization — Beta 5

This release enhances the configuration summary with an override priority section showing which handlers are active and what position each would produce.

### ✨ Features

- **Override Priority section** — displays all 9 pipeline handlers in descending priority order (100→0) with status and position descriptions
  - ✅ **Active** handlers show what position they would set (e.g., "Force Override (100) → 100%")
  - ❌ **Not configured** handlers are clearly marked and skipped at runtime
- **Position descriptions** are context-aware:
  - Fixed positions: `Force Override → 100%`, `Weather Override → 0%`
  - Default positions: `Motion Timeout → default (60%)`, `Cloud Suppression → default (60%)`
  - Calculated positions: `Manual Override → holds calculated position`, `Solar Tracking → calculated (sun position)`, `Climate → calculated (climate strategy)`
  - Glare protection: `Glare Zone → calculated (glare protection)`
- **Always-active handlers** (Manual Override, Solar Tracking, Default) are always shown as active
- **Conditional handlers** (Force Override, Weather, Motion Timeout, Cloud Suppression, Climate, Glare Zone) show active only when relevant sensors/settings are configured
- **Cover-type aware** — Glare Zone only appears for vertical blinds; omitted for awnings and tilts
- Appears in both the initial config flow (pre-creation summary) and the options flow (menu item)
