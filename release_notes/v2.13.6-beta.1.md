# v2.13.6-beta.1

## 🎯 Separate End-Time from Actual Sunset Position

This beta introduces a clearer distinction between two separate sunset-related behaviors that were previously conflated.

### ✨ Features
- **End-time position**: Covers now move to the Default Position when the configured end time is reached (previously moved to Sunset Position)
- **Sunset position**: Covers move to the configured Sunset Position at actual astronomical sunset + offset (now a separate, explicit event)
- **New sunset trigger**: Automatically schedules a one-shot command at actual sunset time to apply the sunset position
- **Edge case handling**: If end time is after sunset, sunset position persists through end time (no revert to default)

### 📝 Configuration Changes
- `return_sunset` toggle (was "Always adjust position to sunset default at end time") is now clearer: "Move covers to default position at end time"
- Config summary now shows two separate lines: `🏠 After end time → default` and `🌅 After sunset → sunset_pos`
- Documentation updated to clarify the distinction

### 🧪 Testing
- Full test suite rewritten for new behavior (1052 tests passing)
- Added 22 new tests for sunset trigger scheduling and end-time behavior
- Comprehensive edge case coverage

### ⚠️ Breaking Changes
**None** - Behavioral change is internal; users with sunset_position configured will see different timing of position changes, but the cover positions themselves remain correct.

### 🔧 Technical Details
- `async_schedule_sunset_trigger()` schedules trigger at actual sunset + offset
- `async_sunset_refresh()` fires at scheduled time, commands sunset_pos with force=True
- Outside time window logic now checks `cover.sunset_valid` to distinguish pre/post-sunset
- Sunset listener cancelled on shutdown

- Diagnostics builder reflects new two-position logic
