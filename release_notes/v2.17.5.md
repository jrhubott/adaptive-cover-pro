## 🎯 All Changes Since v2.17.3

> **Note:** v2.17.4 was a compromised release — the published build was missing the bug fixes described in its own release notes. v2.17.5 supersedes it and contains all fixes that v2.17.4 claimed to ship.

### 🐛 Bug Fixes

- **GlareZoneHandler selected wrong zone (#213):** The glare zone handler was using `max()` instead of `min()` when picking the active zone from multiple configured zones. Since zones closer to the window require more blind coverage to block direct sunlight, the handler should select the *closest* zone — not the farthest. This inversion caused consistent underprotection against glare. Thanks to @ZamenWolk for the analysis. (#214)

- **Cloud/climate sensors didn't trigger immediate updates (#105):** `CONF_CLOUD_COVERAGE_ENTITY`, `CONF_LUX_ENTITY`, `CONF_IRRADIANCE_ENTITY`, and `CONF_OUTSIDETEMP_ENTITY` were never registered with `async_track_state_change_event`. When these sensors changed value, the coordinator only re-evaluated on the next scheduled tick (every 5 minutes by default). Cloud suppression and climate positioning now respond immediately when a sensor crosses its configured threshold.

- **Sun-only position limits incorrectly clamped cloud and climate positions (#105):** Position limits configured as "during sun tracking only" (`enable_min_position` / `enable_max_position` = True) were being applied to cloud suppression and climate handler outputs. These handlers now use the full configured position range; sun-tracking limits apply exclusively to the solar handler.

- **Solar calculation: sill height could produce negative effective distance:** Added a floor guard at 0 after sill height subtraction so negative intermediate values no longer appear in debug logs. The final clipped result was already correct; this eliminates confusing log output.

### ✨ Features

- **Cover Entities and Device Association merged into one screen:** The two-step setup flow has been combined into a single form. Entity selection and device association now appear together — if associated physical devices are found, the device selector is appended to the same screen. The separate "Device Association" menu item in the options flow has been removed. Step renamed to "Covers & Device."

- **Cover capability warnings surface in config summary:** The configuration summary now shows a "Cover Warnings" section when any issues are detected (open/close-only covers, assumed_state, mixed capabilities across entities, sensor-type mismatches, or position limits that will be ignored). The section is omitted for clean configurations.
