## 🎯 Architecture Rewrite — Final Polish

Completes all remaining items from the integration rewrite design spec.

### ✨ New Features

- **Decision Trace Sensor** — New diagnostic sensor showing the full pipeline evaluation chain. State is the winning handler name; attributes include the complete trace with each handler's evaluation, sun position data, and field-of-view validity.
- **Enriched Cover Position Sensor** — Now includes `control_method`, `reason`, `position_explanation`, and `raw_calculated_position` as attributes.
- **Venetian Cover Calculation Engine** — `VenetianCoverCalculation` with `DualAxisResult` for dual-axis covers that control both position and tilt (#33). Calculation engine ready for when config UI support is added.
- **Wind & Cloud Override Stubs** — Pipeline handlers for wind override (priority 90, #28) and cloud suppression (priority 60, #31) ready for implementation.

### 🏗️ Architecture Improvements

- **Typed Configuration** — `CoverConfig`, `VerticalConfig`, `HorizontalConfig`, `TiltConfig` frozen dataclasses replace 22 fragile positional arguments.
- **SunGeometry Engine** — Pure `SunGeometry` class in `engine/sun_geometry.py` extracts all sun position analysis from cover classes.
- **Unified State Snapshot** — `CoverStateSnapshot`, `SunSnapshot`, `CoverCapabilities` frozen dataclasses + `CoverProvider` capture all HA state at the start of each update cycle.
- **Coordinator Slimmed Further** — 1,477 lines (was 1,839 in beta.10, originally 2,700).

### 🧹 Cleanup

- **3 Redundant Sensors Removed** — "Control Method", "Calculated Position", and "Position Explanation" sensors removed. Their data is now available as attributes on the Cover Position sensor. Existing installations auto-cleanup on restart.
- **test_env.ipynb Deleted** — Superseded by `simulate_cover.ipynb`.
- **simulate_cover.ipynb Updated** — Uses `CoverConfig` typed dataclasses.

### 📊 Metrics

- **751 tests** (was 679 in beta.10), **61% coverage** (was 58%)
- `calculation.py` + `sun.py`: 0 Home Assistant imports
- 34 of 35 original spec requirements fully implemented

### 🧪 Testing

- Tested with Python 3.11 and 3.12
- Home Assistant 2024.5.0+
- Backward compatible — existing configurations work without changes

### ⚠️ Beta Notice

This is a pre-release. The 3 removed diagnostic sensors will be auto-cleaned from existing installations on restart. If you relied on the "Control Method" or "Calculated Position" sensors in automations, use the Cover Position sensor's `control_method` and `raw_calculated_position` attributes instead.
