## 🎯 Debug Logging, Position Explanation Sensor, Diagnostic Sensor Cleanup, Duplicate/Sync Covers, Simulation Notebook & Bug Fixes

### ✨ Features

- **Comprehensive debug logging** — Enable `custom_components.adaptive_cover_pro: debug` in your logger config to get detailed logs at every decision point: position explanation changes, why cover moves were skipped (time window, delta, manual override), entity state changes with old/new values, sun tracking transitions (ON→OFF and OFF→ON), climate strategy selection, motion timeout events, and full geometric calculation details (elevation, gamma, effective distance, safety margin, result). Each log entry is prefixed with the device name for multi-instance setups.

- **Position Explanation sensor** — New diagnostic sensor whose *state* is the full position decision string (e.g. `"Sun tracking (45%) → Climate: Winter Heating → 100%"`). Because the state is recorded in HA history, you can now open the History panel and see exactly how decisions changed over time — without needing debug logs.

- **Last Skipped Action sensor** — New diagnostic sensor that records why the most recent automatic cover move was suppressed. State shows the reason (`"Outside time window"`, `"Position delta too small"`, `"Time delta too small"`, `"Manual override active"`). Attributes include the affected entity, the would-be position, and timestamp. Answers the #1 troubleshooting question: "why didn't my cover move?"

- **Calculation detail attributes on Calculated Position sensor** — Five new attributes on the Calculated Position sensor (vertical covers only): `edge_case_detected`, `safety_margin`, `effective_distance`, `window_depth_contribution`, `sill_height_offset`. Shows exactly what the geometric calculation did.

- **Position Explanation diagnostics (Issue #68)** — `position_explanation` attribute on the Calculated Position sensor shows a human-readable decision chain. New `climate_strategy` attribute shows which climate strategy branch was applied.

- **Duplicate and sync covers in config flow (Issue #54)** — "Duplicate existing cover" in the Add Integration flow copies all settings from a source cover. "Sync settings to other covers" in Options pushes shared settings to multiple covers of the same type.

- **Diagnostic sensor consolidation** — Removed the "Enable diagnostic entities" config toggle. All devices now always get diagnostic sensors. Consolidated 17 sensors into 9 cleaner sensors:
  - **Sun Position** — azimuth (state), elevation, gamma, FOV bounds, in_fov, blind spot as attributes
  - **Control Status** — status + reason + time window + sun validity; delta debug attributes
  - **Calculated Position** — raw position + decision chain + calculation details
  - **Position Explanation** — decision chain as state (new, for HA history)
  - **Last Skipped Action** — why the last move was skipped (new)
  - **Position Verification** — combines verification + retry count
  - **Motion Status** — combines timeout end + last motion time (when configured)
  - **Climate Status** — active temperature + conditions (when climate mode enabled)
  - Upgrade automatically removes old entity registry entries

- **Simulation notebook** — `notebooks/simulate_cover.ipynb` visualizes cover behavior for any date range using your actual HA config. Export via **Developer Tools → Services → Adaptive Cover Pro: Export Configuration**, paste into notebook, run to generate sun/cover/FOV plots.

### 🐛 Bug Fixes

- **Position Explanation and Last Skipped Action sensors showing Unavailable (beta.8)** — Removed `native_unit_of_measurement = ""` from both new sensors. Setting an empty string unit (rather than omitting it) caused HA to treat them as numeric sensors and fail to write string state values, marking them unavailable.
- **Motion Status sensor shows "Unknown" most of the time (Issue #75)** — Now displays `"Motion Detected"`, `"Timeout Pending"`, `"No Motion"`, `"Waiting for Data"` instead of timestamps or None.
- **export_config service TypeError** — Fixed incorrect `(hass, call)` service handler signature; now uses `call.hass`.
- **Control Status crash with offset-naive timestamp** — `seconds_since_last_action` TypeError when timestamp lacked timezone info; now treats naive timestamps as UTC.
- **Position sensor reports stale value after end_time (Issue #66)** — Now correctly shows sunset/default position when outside the configured time window.
- **Help and Known Issues links point to correct repo (Issue #67)** — Updated repository URLs.
- **Fixed double-instantiation of ClimateCoverState** — `climate_mode_data()` now reuses a single instance.
- **Sunset Position slider correctly handles empty/None values** — Changed from BOX to SLIDER mode so clearing saves `None` instead of `0`.
- **position_explanation correctly distinguishes Sunset Position vs Default Position** — Shows `Default Position` when no sunset position is configured.

### ⚠️ Breaking Change (Diagnostic Sensors)

Users who had `enable_diagnostics: true` will see old diagnostic entities become unavailable, replaced by new consolidated entities. The upgrade automatically removes old entries from the entity registry. Update any automations or dashboard cards referencing old entity IDs (e.g. `sensor.my_cover_sun_azimuth`) to use the new consolidated sensors (e.g. `sensor.my_cover_sun_position`).
