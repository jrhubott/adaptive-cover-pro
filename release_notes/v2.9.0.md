# v2.9.0 — Config Flow Redesign

## ⚠️ Heads Up for Existing Users

This release includes a significant redesign of the setup and configuration flow. While your existing configurations will continue to work without any changes, you may notice that the **options screens look quite different** when you next open them.

We sincerely apologize for any inconvenience this redesign causes. The old layout had grown difficult to navigate — one screen could have 22 fields at once, and important settings like Climate were impossible to enable after initial setup. This rewrite organizes everything into focused, logical sections that will make ongoing configuration much easier and allow us to add new features more cleanly going forward.

**If you are migrating from the original AdaptiveCover integration:** we have removed the automatic import wizard. You will need to recreate your configurations manually. We're sorry for the extra work — this lets us move forward without maintaining compatibility with the old integration's data format.

---

## ✨ What Changed

### Config Flow Redesign

The old single-screen-per-cover-type layout (up to 22 fields!) has been replaced with 10 focused steps, each covering one concept:

| Step | What it covers |
|------|---------------|
| **Cover Entities** | Entity selection and device association |
| **Geometry** | Cover dimensions (type-specific: no more `window_depth` on awnings) |
| **Sun Tracking** | Azimuth, field of view, elevation limits |
| **Position** | Default position, limits, sunset behavior, offsets |
| **Automation** | Delta thresholds, schedule (start/end time) |
| **Manual Override** | Override duration, reset behavior, threshold |
| **Motion & Force Overrides** | Motion sensors, force override sensors |
| **Climate** | Climate mode toggle + all climate settings |
| **Weather** | Weather entity (shown when set) |
| **Update Interval** | Coordinator refresh rate |

### Options Menu Now Always Shows Climate

Previously, the Climate option was hidden in the options menu unless Climate Mode was already enabled — making it impossible to turn on Climate Mode after the initial setup without reinstalling. This is now fixed: **Climate is always visible** in the options menu.

### Options Menu Mirrors Initial Setup

The options menu now lists exactly the same sections as the initial setup flow, making it easy to find and update any setting.

### Bug Fixes

- Fixed an issue where the import counter could show incorrect values
- Fixed list entity validation for force override and motion sensor fields
- Fixed the entity selector for tilt covers not filtering to tilt-capable devices
- Fixed the interpolation options error path in the options flow
