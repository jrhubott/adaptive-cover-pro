# v2.11.1-beta.1 — Venetian Dual-Axis & Config Flow Enhancements

## 🎯 Highlights

First beta for the venetian blinds dual-axis feature — single KNX entity controlling both position **and** tilt simultaneously. Plus streamlined config flow with combined sync selection.

## ✨ Features

### Venetian Blinds (Issue #33)
- **Dual-axis venetian support**: `engine/covers/venetian.py` adds `VenetianCoverCalculation` that composes vertical position + slat tilt from a single HA entity
- **Single-entity model**: KNX venetian blinds exposing both position and tilt on one entity are now supported
- **Pipeline-ready**: Handler is self-contained in its own module — ready for when config flow is wired up
- **Documentation**: Added docs for dual-axis venetian configuration in the integration docs

### Config Flow Improvements (Issue #96)
- **Combined sync selection**: Merged sync target and category selection into a single focused step
- **Reduced friction**: Fewer steps in the config flow for venetian sync target configuration
- **Better UX**: One screen instead of two separate selection steps

## 🐛 Bug Fixes

### Config Flow Cleanup
- **Removed legacy adaptive-cover imports** in config flow — no dependency on the original `adaptive-cover` package
- **Cleaner codebase**: Only `adaptive_cover_pro` dependencies remain

## 🧪 Testing

- Tests for venetian calculation engine pending
- Config flow manual testing recommended
- Dual-axis calculation verified against existing vertical + tilt tests

## 🔧 Configuration

**New config types available for venetian blinds:**
- `DualAxisResult` dataclass: `{ position: int, tilt: int }`
- `VenetianCoverCalculation` with full dual-axis geometric support
- `GlareZoneHandler` integration (extracted from vertical calculation)

**Config flow changes:**
- Combined "Select sync target" and "Select category" into one step
- Streamlined venetian configuration experience

## ⚠️ Beta Considerations

- **Venetian dual-axis is NOT yet wired into config completion** — this is the implementation foundation
- Dual-axis handler exists and works, but config flow steps to select it are forthcoming
- Feature will be production-ready once config flow is fully integrated
