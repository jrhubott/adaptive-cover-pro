## 🎯 Position Explanation Diagnostics, Diagnostic Sensor Cleanup, Duplicate/Sync Covers, Simulation Notebook & Bug Fixes

### ✨ Features
- **Position Explanation diagnostics (Issue #68)** — New `position_explanation` attribute on the Calculated Position sensor shows a human-readable decision chain explaining exactly why the cover is at its current position (e.g. `"Sun tracking (45%) → Climate: Winter Heating → 100%"`). New `climate_strategy` attribute shows which climate strategy branch was applied (`winter_heating`, `summer_cooling`, `low_light`, `glare_control`). Helps diagnose unexpected cover positions, especially when climate mode overrides the calculated position.
- **Duplicate and sync covers in config flow (Issue #54)** — Add "Duplicate existing cover" option to the Add Integration flow: select a source cover, enter only name/entities/azimuth — all other settings are copied automatically. Add "Sync settings to other covers" option to every cover's Options flow: select one or more covers of the same type and push all shared settings in one click.
- **Diagnostic sensor consolidation** — Removed the "Enable diagnostic entities" config toggle. All devices now always get diagnostic sensors — no setup required. Consolidated 17 diagnostic sensors into 9 cleaner sensors by combining related sensors:
  - **Sun Position** — azimuth (state), plus elevation, gamma, FOV bounds, in_fov, blind spot range as attributes
  - **Control Status** — combines Control Status + Control State Reason + Time Window Status + Sun Validity Status. New debug attributes: `delta_position_threshold`, `delta_time_threshold_minutes`, `position_delta_from_last_action`, `seconds_since_last_action`, `last_updated`
  - **Calculated Position** — enriched with new `sun_calculated_position` attribute showing the raw sun-geometry position before climate/limits/inverse adjustments
  - **Position Verification** — combines Last Position Verification + Position Verification Retries into a single sensor
  - **Motion Status** — combines Motion Timeout End Time + Last Motion Time (when motion sensors configured)
  - **Climate Status** — combines Active Temperature + Climate Conditions (when climate mode enabled)
  - Upgrade automatically removes old entity registry entries — no manual cleanup needed
- **Simulation notebook** — New `notebooks/simulate_cover.ipynb` lets you visualize cover behavior for any date or date range using your actual Home Assistant configuration. Export your cover config via **Developer Tools → Services → Adaptive Cover Pro: Export Configuration**, paste the JSON into the notebook, set a date range, and run to generate plots showing sun position, cover position, and FOV window. Supports single-day and multi-day overlay views.

### 🐛 Bug Fixes
- **export_config service TypeError** — Service handler used incorrect `(hass, call)` signature; HA only passes `ServiceCall`. Fixed to use `call.hass` instead.
- **Control Status crash with offset-naive timestamp** — `seconds_since_last_action` in the Control Status diagnostic caused a `TypeError` when the stored `last_cover_action` timestamp lacked timezone info. Now correctly treats naive timestamps as UTC before computing elapsed time.
- **Position sensor reports stale value after end_time (Issue #66)** — After end_time passed, the sensor reported the sun-tracked value while the cover was already commanded to the sunset/default position. Now correctly shows the sunset/default position when outside the configured time window.
- **Help and Known Issues links point to correct repo (Issue #67)** — Updated links to point to the correct repository URL.
- **Fixed double-instantiation of ClimateCoverState** — `climate_mode_data()` previously created two `ClimateCoverState` instances unnecessarily; now reuses a single instance.
- **Sunset Position slider now correctly handles empty/None values** — Changed from BOX to SLIDER mode so clearing the field correctly saves `None` instead of `0`.
- **position_explanation correctly distinguishes Sunset Position vs Default Position** — When no sunset position is configured, the explanation now shows `Default Position` instead of `Sunset Position`.

### ⚠️ Breaking Change (Diagnostic Sensors)
Users who had `enable_diagnostics: true` will see their old diagnostic entities become unavailable and replaced by new consolidated entities. The upgrade automatically removes the old entities from the entity registry. If you have automations or dashboard cards referencing the old entity IDs (e.g. `sensor.my_cover_sun_azimuth`, `sensor.my_cover_sun_elevation`, `sensor.my_cover_gamma`), update them to use the new consolidated entities (e.g. `sensor.my_cover_sun_position`).
