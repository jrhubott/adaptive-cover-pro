## 🎯 Companion Lovelace Card, Dimension Standardization, and Reliability Fixes

### ✨ Features

- **New companion Lovelace card — [adaptive-cover-pro-card](https://github.com/jrhubott/adaptive-cover-pro-card):**
  A custom Lovelace card purpose-built for Adaptive Cover Pro is now available.
  One glance tells you what the integration is doing, why it decided that, and how
  to take over. It visualizes live sun position vs. window FOV and blind spot,
  renders the full pipeline decision trace with handler priorities, shows live cover
  positions, and provides inline manual override controls. HACS-installable as a
  Lovelace plugin — requires no integration changes, reads existing entities, and
  calls existing services. See the
  [Lovelace Card wiki page](https://github.com/jrhubott/adaptive-cover-pro/wiki/Lovelace-Card)
  for install, configuration, and usage.

- **Standardize dimensions on metres; auto-migrate existing configs (#251):**
  All cover dimension settings (height, awn_length, window_depth, sill_height) are
  now stored and displayed in metres. Existing configurations are automatically
  migrated on the next HA restart — no manual action required.

- **Config summary redesign — priority-annotated decision chain (#245):**
  The configuration summary shown at the end of setup and options flows has been
  redesigned. The Position Map section is removed (it duplicated rules already
  visible in How It Decides), each decision-chain rule now shows a `[priority]`
  badge so the override order is visible at a glance, and a new **Today's Sun**
  block shows raw and effective sunrise/sunset times alongside the solar control
  window. Fourteen previously-hidden behavior-affecting options are now surfaced
  in the summary.

- **Service descriptions translated into German and French (#261):**
  Service names and descriptions are now fully translated in `de.json` and `fr.json`,
  matching the coverage of all other user-visible strings.

### 🐛 Bug Fixes

- **Cloud suppression now reads lux/irradiance regardless of Climate Mode toggle (#268):**
  When Climate Mode was disabled, cloud suppression silently ignored configured lux
  and irradiance sensors because the reads were gated on the climate-mode toggle
  switches, which only exist when Climate Mode is on. `use_lux`/`use_irradiance` are
  now also set True when cloud suppression is enabled with the matching entity
  configured, regardless of the Climate Mode toggle state. Climate Mode behavior is
  unchanged — ClimateHandler already gates itself via `climate_mode_enabled`.

- **Covers now reposition to sunset position when end time is before astronomical sunset (#266):**
  When the configured end time fell before sunset + sunset_offset, the integration
  correctly sent the daytime default at end time (the sunset window wasn't open yet),
  but never followed up when the sunset window later opened — leaving covers at 100%
  all night. The coordinator now detects the `is_sunset_active` False→True transition
  each minute and dispatches the sunset position at that moment.

- **Min-mode floor uses default position when sun tracking is off (#264):**
  When sun tracking was disabled, the min-mode floor was using an incorrect fallback
  value. It now correctly uses the configured default position.

- **Weather override self-heals when stuck after missed events (#255):**
  The weather override could become permanently stuck if HA missed the state-change
  event that should have cleared it (e.g., after a restart or during HA load spikes).
  The integration now validates the override condition on every update cycle and
  clears it automatically when conditions are no longer met.

- **Weather override restored before first pipeline run after restart (#255):**
  After a HA restart, the first update cycle could dispatch a solar-tracked position
  before the weather override was re-applied, causing a brief one-cycle flicker to
  the solar position even while weather conditions were still active. The recovery
  call is now made earlier in the update flow so the pipeline always sees the
  restored state on the very first cycle.

- **Horizontal awning extension now capped at awn_length (#209):**
  The computed extension distance could exceed the configured awning length, sending
  a move command beyond the cover's physical limit. Extension is now saturated at
  `awn_length`.

### 📖 Documentation

- **Glare zone setup guide diagram:** Added a visual diagram to the glare zone
  configuration guide showing coordinate conventions and angle relationships.

- **Clarify Position Calibration input/output labels (#254):**
  The Position Calibration step labels were ambiguous about whether the input
  was the raw sensor reading or the desired HA position. Labels now explicitly
  identify each column.

- **Clarify blind spot angles are measured from FOV left boundary (#211):**
  The config flow hint for blind spot start/end angles now states that angles are
  measured from the left edge of the field of view, not from azimuth north.

- **README slimmed to quickstart scope (#249):**
  Architecture details, feature lists, and reference content have moved to the
  Wiki. The README now covers badges, highlights, 30-second install, and links
  to Wiki pages.
