## 🎯 New Diagnostic Sensors for Motion Control and Force Override

### ✨ Features

- **Motion Timeout End Time** sensor (P0, enabled by default) — shows the exact timestamp when the motion timeout will fire and covers will return to the default position. Only created when motion sensors are configured. Answers "when will my covers auto-close after I leave the room?"
- **Manual Override End Time** sensor (P0, enabled by default) — shows when the current manual override will expire and automatic control resumes. Per-entity expiry times in attributes for multi-cover setups.
- **Force Override Triggers** sensor (P1, disabled by default) — counts how many force override sensors are currently active, with per-sensor state (`on`/`off`/`unavailable`) as attributes. Only created when force override sensors are configured.
- **Last Motion Time** sensor (P1, disabled by default) — timestamps the last motion detection event for debugging motion control behavior. Only created when motion sensors are configured.

### 🔧 Changes

- Default manual override duration increased from 15 minutes to 2 hours
