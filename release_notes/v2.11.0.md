# v2.11.0 — Cloud Coverage Sensor Support

## 🎯 Highlights

Adds support for cloud coverage percentage sensors (0-100%) as a precise alternative to categorical weather conditions for glare control suppression.

## ✨ Features

- **Cloud coverage sensor support** (Issue #94): Configure a percentage-based cloud coverage sensor (e.g., from Google Weather) as a signal for the "Suppress glare control in low light" feature. When the sensor value reaches or exceeds the configured threshold (default: 75%), glare control is suppressed and covers return to their default position.
- **Cloud suppression pipeline registration**: The cloud suppression handler is now fully active in the override pipeline at priority 60 (between manual override and climate mode).

## 🛠️ Configuration

Two new optional fields in Climate Settings:

- **Cloud coverage sensor** — any sensor reporting cloud coverage as 0-100%
- **Cloud coverage threshold** — percentage at or above which glare control is suppressed (default: 75%)

Cloud coverage uses OR logic alongside the existing weather entity, lux sensor, and irradiance sensor — any one signal indicating no direct sun activates cloud suppression.
