## 🐛 Fix: Device association "None" not clearing; orphaned standalone device

### Bug Fixes

- **"None (standalone device)" now reliably clears the association** — The device selector previously used an empty string `""` for the "None" option, which HA's form handling could silently discard, causing the old association to be restored. The selector now uses a non-ambiguous sentinel value so clearing the association always works.

- **Orphaned standalone device removed after association** — When an Adaptive Cover instance is associated with a physical device, the old standalone "Adaptive X Cover" virtual device is now explicitly removed from the device registry on reload, so it no longer appears as an extra entry under the integration.

### Notes

These are follow-up fixes to `v2.7.10-beta.2`. If you tested beta.2, reload the integration after updating to apply both fixes. If you had a stale association that wouldn't clear, update to beta.3, open Options → Device Association, select "None (standalone device)", and save.
