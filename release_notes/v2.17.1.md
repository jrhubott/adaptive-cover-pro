## Manual Override on Somfy My Stop + Entity Registration Fix

### ✨ Features

- **Manual override via stop_cover for Somfy RTS covers (#202):** Pressing STOP on a non-position-capable cover (e.g. Somfy RTS) now correctly triggers a manual override. Previously, Somfy RTS covers move to the hardware "My" preset when STOP is pressed, but never report a new position — so the integration had no way to detect the user's intent and would fight the preset on the next solar calculation. ACP now listens for `cover.stop_cover` service calls on tracked covers and, when a configured "My position" value is set, immediately flags the cover as manually overridden at that position. Integration-originated stop calls (e.g. from ACP's own safety handlers) are filtered out so they don't falsely trigger an override.

### 🐛 Bug Fixes

- **Entity registration race condition on reload (#203):** On some installs, reloading a cover config entry caused up to 8 entities (Sun Infront, Manual Override, Control Status, Climate Status, Position Verification, and others) to fail registration with `AttributeError` and become unavailable for the session. Root cause: PR #195 reordered setup so platforms are forwarded before `first_refresh`, but `async_add_entities` schedules entity registration as background tasks. If those tasks wrote state before `first_refresh` populated `coordinator.data`, properties reading `coordinator.data.states` or `coordinator.entities` raised. Fixed by initialising `coordinator.entities` at construction time and adding `available` guards that return `False` (rendering the entity unavailable rather than crashing) until `coordinator.data` is populated. Entities become available within milliseconds once the first refresh completes.

### 🧪 Testing
