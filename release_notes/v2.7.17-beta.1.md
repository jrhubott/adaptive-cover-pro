## 🎯 Position Explanation Diagnostics, Duplicate/Sync Covers & Bug Fixes

### ✨ Features

- **Position Explanation diagnostics (Issue #68)** — New `position_explanation` attribute on the Calculated Position sensor shows a human-readable decision chain explaining exactly why the cover is at its current position (e.g. `"Sun tracking (45%) → Climate: Winter Heating → 100%"`). New `climate_strategy` attribute shows which climate strategy branch was applied (`winter_heating`, `summer_cooling`, `low_light`, `glare_control`). Helps diagnose unexpected cover positions, especially when climate mode overrides the calculated position.
- **Duplicate and sync covers in config flow (Issue #54)** — Add "Duplicate existing cover" option to the Add Integration flow: select a source cover, enter only name/entities/azimuth — all other settings are copied automatically. Add "Sync settings to other covers" option to every cover's Options flow: select one or more covers of the same type and push all shared settings in one click.

### 🐛 Bug Fixes

- **Position sensor reports stale value after end_time (Issue #66)** — After end_time passed, the sensor reported the sun-tracked value while the cover was already commanded to the sunset/default position. Now correctly shows the sunset/default position when outside the configured time window.
- **Help and Known Issues links point to correct repo (Issue #67)** — Updated links to point to the correct repository URL.
- **Fixed double-instantiation of ClimateCoverState** — `climate_mode_data()` previously created two `ClimateCoverState` instances unnecessarily; now reuses a single instance.
