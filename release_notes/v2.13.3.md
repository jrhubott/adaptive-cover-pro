# Adaptive Cover Pro ⛅ v2.13.3

## 🐛 Bug Fixes

### Crash Protection — 16 Defensive Hardening Fixes

This release includes a comprehensive set of defensive fixes to eliminate crashes and edge-case failures discovered during a deep code review:

- **Horizontal cover division-by-zero** — guard added when `c_angle ≈ 0` (sin near zero)
- **Tilt cover sqrt(negative)** — discriminant check added; returns `0.0` (closed) safely with NaN guard
- **Vertical cover cos(gamma) near-zero** — clamps to minimum `0.01` to prevent runaway calculations
- **Polar latitude crash** — `SunData.sunset()` / `sunrise()` now catch `ValueError` and return safe sentinel values
- **Sensor float conversion errors** — lux, irradiance, cloud coverage, and temperature entity reads now wrapped in `try/except` with debug logging instead of crashing
- **None position coalescing** — `pos_sun` values of `None` are coalesced to `0.0` before passing to the calculation engine
- **Config None defaults** — `CoverConfig`, `VerticalConfig`, and `HorizontalConfig` now apply safe defaults for any `None` option values at startup
- **Coordinator init** — `default_state` initialized to `0` and `normal_cover_state` to `None` to prevent attribute access before first update
- **Manual override** — `handle_state_change` returns early when `new_position` is `None`
- **asyncio.CancelledError** — caught in all four timeout handlers (motion, weather, and both grace period timers)
- **Cover service call guard** — `hass.services.async_call` in `cover_command.py` now wrapped with `HomeAssistantError` handler
- **Deprecated `datetime.utcnow()`** — replaced with `datetime.now(UTC)` throughout

### Config Flow Fixes

- **Fix #109** — Config sync now saves the current cover's settings _before_ extracting shared options, ensuring latest values are always carried over; returns to the options menu after syncing instead of aborting
- **Fix #108** — Sync UX defaults (confirm toggle now defaults to off, requiring explicit confirmation); all position descriptions standardized to cover-type-aware phrasing (`0% = closed / 100% = open`)

## 🔧 Internal Improvements

- **`CoverCommandService` redesigned** as a fully self-contained positioning service with a built-in reconciliation timer — eliminates the need for `PositionVerificationManager`
- **`PositionVerificationManager` removed** — its responsibilities (retry tracking, target-reached detection) are now handled directly by `CoverCommandService.apply_position()` and `_reconcile()`
- **All cover positioning unified** through a single `apply_position()` entry point
- **Sun-just-appeared transition** now fires exactly once per update cycle regardless of entity count (previously could fire once per entity)
- **Reconciliation log spam fix** — warning logged once when max retries hit, then degrades to debug; cleared when a new target is set
