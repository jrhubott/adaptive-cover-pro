## 🎯 Glare Zones — Floor-Area Shading for Vertical Blinds

### ✨ Features

- **Glare Zones (Issue #64):** Define up to 4 named floor zones per vertical blind. Each zone is a circle on the floor (x/y position + radius in cm). The blind automatically lowers far enough to shade every active zone based on the current sun angle — no more manually dialing in a single depth number.

- **Per-zone switch entities:** Each configured zone gets its own switch (`switch.*_glare_zone_<name>`). Toggle zones on/off at runtime via HA automations or manually — for example, disable the "Desk" zone on weekends.

- **Glare Active binary sensor:** `binary_sensor.*_glare_active` turns on when a glare zone is actually extending the blind position beyond the base distance. Useful for automations and dashboards ("glare is hitting something right now").

- **Top-level options menu:** Glare Zones appears as a permanent menu item when reconfiguring a vertical blind — no need to go through the geometry step to adjust zones.

### 🔧 How It Works

Zones use a coordinate system centred on the window:
- **X axis:** Along the wall (positive = right when facing the window from inside), cm
- **Y axis:** Into the room (perpendicular to the window), cm
- **Radius:** Protection circle around the zone centre, cm

At each update cycle, the integration calculates which zones can receive sunlight through the window at the current sun angle, finds the maximum required blind depth across all active zones and the existing base distance, and sets the blind accordingly. Existing installations with no zones configured are completely unaffected.

### 🧪 Testing

This beta covers vertical blinds only. Please test:

1. Configure a vertical blind and go to **Settings → Integrations → your blind → Configure → Glare Zones**
2. Enable glare zones, set window width, and add at least one zone (e.g. "Desk" at X=0, Y=200cm, Radius=30cm)
3. Verify `switch.*_glare_zone_desk` appears and defaults to ON
4. Verify `binary_sensor.*_glare_active` appears
5. When the sun is in the blind's field of view, verify the cover position is deeper than it would be with just the base distance
6. Toggle the zone switch OFF — cover should revert to base-distance position
7. Test with multiple zones, including non-contiguous slots (e.g. zones 1 and 3, leave zone 2 blank)

### 📋 Installation

Update via HACS or copy `custom_components/adaptive_cover_pro/` to your config directory and restart Home Assistant.
