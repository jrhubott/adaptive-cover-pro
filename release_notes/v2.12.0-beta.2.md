## 🔧 Pipeline Plugin Architecture — Beta 2

This release continues the pipeline plugin architecture refactoring (beta.1 introduced the initial refactor; beta.2 addresses post-review issues).

### 🐛 Bug Fixes

- Fixed `GlareZoneHandler` diagnostic reason string when multiple glare zones share the maximum effective distance — all contributing zones now appear correctly
- Removed dead `active_zone_names` computation in coordinator (was computed twice; only the snapshot copy is needed)
- Fixed `ClimateCoverData` carrying a vestigial `logger` field that served no function in the new handler

### ✨ Improvements

- `ClimateOptions` and `PipelineSnapshot` are now frozen dataclasses, enforcing the read-only contract that the plugin architecture depends on
- `active_zone_names` field typed as `frozenset[str]` for consistency with the frozen snapshot

### 🧪 Testing

- Added test for two glare zones with equal maximum distance — both zones appear in the reason string
- Added test for one naturally-blocked zone alongside a valid zone — handler correctly uses the valid zone
- 856 tests passing

### ⚙️ Technical

- Requires Home Assistant 2024.5.0+
- Tested with Python 3.11 and 3.12
