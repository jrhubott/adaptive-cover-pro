## 🐛 Bug Fix

### Fix: Motion Status sensor crashes all entities on startup (#76)

The Motion Status sensor had `_attr_native_unit_of_measurement = ""` which caused Home Assistant to treat it as a numeric sensor. When the sensor returned string states like `"waiting_for_data"`, HA threw a `ValueError` during entity registration, preventing **all** integration entities (sensors, switches) from loading.

This is the same fix applied in v2.7.17-beta.8 for the Position Explanation and Last Skipped Action sensors — the Motion Status sensor was missed in that pass.
