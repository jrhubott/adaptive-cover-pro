## 🐛 Bug Fix Release

### 🐛 Bug Fixes

- **Covers no longer move outside the time window on HA restart** (#179) — `async_handle_first_refresh` now checks the time window before commanding covers. If Home Assistant restarts at midnight or another time outside the configured window, covers will no longer be repositioned until the window opens. The reconciliation timer also now skips non-safety targets when outside the window, and stale daytime targets are cleared when the window closes.

- **Covers no longer move outside the time window at sunrise** (#173) — The coordinator was sending commands on entity state changes before the start time was reached. Added time-window guard to `async_handle_state_change` and `_async_send_after_override_clear`.

- **Custom positions no longer trigger false manual override** (#172) — Integration-initiated moves (from custom position sensors and other automation) were being misdetected as manual user moves, locking out automatic control. Automation moves are now tracked and excluded from manual override detection.

- **Horizontal cover no longer crashes with NoneType on window_depth** (#174) — `options.get(CONF_WINDOW_DEPTH, 0.0)` did not handle an explicit `None` value stored in config; changed to `or 0.0`.

- **Tilt cover position clamped to 0–100%** (#153) — `AdaptiveTiltCover.calculate_position()` could return values above 100% (e.g. 101.4) for very narrow FOV angles combined with high sun elevation. Position is now clamped with `max(0, min(100, position))`.

- **Duplicate unique_id for Manual Override entities resolved** (#154) — The Manual Override binary sensor and switch both generated the same `unique_id`, causing HA to silently merge them in the entity registry. The binary sensor now uses its `key` field for a distinct ID.

- **Diagnostics no longer returns unmarshalable mappingproxy** (#155) — `async_get_config_entry_diagnostics` was returning `config_entry.options` as a raw `mappingproxy`, which fails `json.dumps()` without a custom encoder. Wrapped in `dict()`.
