# v2.13.0 — Safety Overrides, Config Flow Polish, Improved Icons

**Release Date:** 2026-04-03

## 🎯 Overview

v2.13.0 is a major update focusing on **safety, usability, and configuration clarity**. Key additions include automatic bypass of Automatic Control for safety overrides (force override and weather conditions), improved config flow organization, renamed/reorganized settings for clarity, and comprehensive icon updates for all entities.

## 🛡️ Safety Overrides Now Bypass Automatic Control

Force Override and Weather Override now operate **even when Automatic Control is disabled**, ensuring covers remain protected from wind, rain, and other hazards when you've paused automation. This is a critical safety feature for weather protection.

### Implementation
- Added `bypass_auto_control: bool` flag to pipeline result
- Force Override (priority 100) always bypasses automatic control
- Weather Override (priority 90) bypasses by default, but now **configurable**
- New toggle in Weather Override config: **"Active when Automatic Control is off"** (default: on)
- When disabled, weather override follows Automatic Control switch like all other automation
- Decision Trace sensor exposes bypass status
- 23 new tests covering all bypass scenarios

## 📋 Config Flow Improvements

### Position Settings Reorganization
- **Position Calibration** moved directly under Position Settings (was Interpolation, separate menu item)
  - Renamed "Interpolation" → "Position Calibration" for clarity
  - Short label: "Enable Position Calibration (custom open/close position mapping)"
  - Now appears after Position Settings in the options menu

### Menu Label Alignment
- All config menu option names now match their dialog titles
  - "Blind spot configuration" → "Blind Spot"
  - "Range Adjustment" → "Position Calibration"
  - "Climate settings" → "Climate Configuration"
  - Added missing title/description to "Schedule & Timing" (automation step)
  - Applied to all 13 translation files (en, cs, de, es, fr, hu, it, nl, pl, pt-BR, sk, sl, uk)

### Copy Settings Dialog Fixes
- Target covers and categories start with **nothing pre-selected**
- Sync categories remain pre-checked for convenience (only target covers require explicit selection)
- Clearer user intent: no accidental syncs

### Sync Category Enhancements
- Renamed sync labels for clarity:
  - "Window Dimensions" → "Cover Geometry (dimensions, depth, sill height)"
  - "Climate" → "Climate Configuration (temperature, presence, lux)"
- Added missing **Glare Zones** sync category (CONF_ENABLE_GLARE_ZONES + 16 zone keys)
- Added missing **Window Width** to geometry category
- Added **Azimuth Exclusion Notice** to sync dialog (window azimuth cannot be synced between covers)

### Window Width Reorganization
- **Moved from Glare Zones step → Cover Geometry step**
- Logical grouping: window width is a physical dimension, not a glare zone definition
- Backwards compatible: config key unchanged, existing values pre-populated

## 🎨 Entity Icons (Issue #100)

All switches, binary sensors, and buttons now have semantic Material Design Icons:

### Switches
- **Automatic Control** — `mdi:window-shutter-auto` (on) / `mdi:window-shutter` (off) *(Issue #100)*
- **Manual Override** — `mdi:hand-back-left` (on) / `mdi:hand-back-left-off` (off) *(Issue #100)*
- **Return to default** — `mdi:restore` icon *(Issue #100)*
- **Climate Mode** — `mdi:home-thermometer` (on) / `mdi:home-thermometer-outline` (off) *(Fixed: was outline-only)*
- **Outside Temperature** — Added `mdi:thermometer-off` for disabled state
- **Irradiance** — Fixed inverted icons (filled=on, outline=off per MDI convention)
- **Glare Zones 1–4** — `mdi:blinds` (on) / `mdi:blinds-open` (off)

### Binary Sensors
- **Manual Override** — `mdi:hand-back-left` / `mdi:hand-back-left-off` (consistent with switch)
- **Glare Active** — `mdi:white-balance-sunny` (on) / `mdi:weather-sunny-off` (off)
- **Position Mismatch** — `mdi:alert-circle-outline` (problem) / `mdi:check-circle-outline` (ok)

### Buttons
- **Reset Manual Override** — `mdi:hand-back-left-off` (moved to `icons.json` for consistency)

**Technical:** All icons defined in `icons.json` following MDI conventions (filled = active, outline = inactive). Added `_attr_translation_key` to button and diagnostic binary sensors for proper icon resolution.
