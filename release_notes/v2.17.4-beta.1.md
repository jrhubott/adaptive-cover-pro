## Fix: Cloud suppression now fires immediately when sensor changes

### Bug Fixes

- **Cloud suppression delayed response (#105):** Cloud coverage, lux, irradiance, and outside temperature sensors were not registered for state change tracking. When these sensors crossed their configured thresholds, the integration did not immediately re-evaluate the pipeline — it waited for the next periodic refresh. Cloud suppression (and climate mode) now responds instantly when any of these sensor values change.
