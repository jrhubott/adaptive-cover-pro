## 🎯 Optional Device Association

### ✨ Features
- **Device Association** — Link integration entities (sensors, switches, buttons) to an existing physical device (e.g., a blind motor) so they appear directly under that device instead of a standalone virtual service device
- Available during initial setup (auto-derived from controlled cover entities) and anytime via the options flow menu
- Easily switch back to standalone mode or change the associated device

### 🐛 Bug Fixes
- Fix orphaned standalone device remaining after activating device association
- Fix clearing device association not reliably removing stale config entry links from the physical device
- Use non-empty sentinel for standalone option to prevent HA form handling from silently restoring a stale default

### 🧪 Testing
- 11 new tests for device association (helper function, device_info merging/fallback, config storage, options flow)
- Tested with Python 3.11 and 3.12
- Home Assistant 2024.5.0+
