## 🐛 Bug Fix: Control Method Sensor Display

### 🐛 Bug Fixes

- **Control Method sensor** no longer shows `ControlMethod.DEFAULT` (or similar enum repr) — it now correctly displays the string value (`default`, `solar`, `summer`, `winter`, `manual_override`, `motion_timeout`, `force_override`)

### 🔍 Root Cause

Python 3.11+ changed `str()` behavior on `str, Enum` subclasses. Previously `str(ControlMethod.DEFAULT)` returned `"default"`; in 3.11+ it returns `"ControlMethod.DEFAULT"`. Home Assistant's state machine calls `str()` when persisting sensor states, causing the raw enum repr to be shown. Fixed by using `.value` explicitly when storing the control method in the coordinator states dict.
