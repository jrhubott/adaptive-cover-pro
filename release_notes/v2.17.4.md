## Glare Zone Selection Fix + Cloud Suppression Responsiveness

### 🐛 Bug Fixes

- **GlareZoneHandler inverted zone selection logic (#213):** The glare zone handler was using `max()` instead of `min()` when selecting the active zone from multiple configured zones. Since zones closer to the window require more blind coverage to block direct sunlight, we should select the zone with the *minimum* distance, not maximum. This inversion caused the handler to consistently underprotect against glare by selecting the farthest zone instead of the closest one. (Thanks @ZamenWolk)
- **Cloud suppression now fires immediately on state changes:** Previously, cloud/climate sensor state changes were only observed during the next scheduled refresh cycle (every 5 minutes by default). Cloud suppression and climate positions now update instantly when a sensor state changes, dramatically improving responsiveness in variable cloud conditions.
- **Cloud/climate positions no longer clamped by sun-only limits (#105):** Position limits configured as "during sun tracking only" (`enable_min_position` / `enable_max_position` = True) were incorrectly applied to cloud suppression and climate positioning decisions. These handlers now respect the full configured position range. Sun-tracking limits apply only to the solar handler.
