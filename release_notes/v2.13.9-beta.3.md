# v2.13.9-beta.3 — Custom Positions Handler

## 🎯 Custom Positions Feature (New)

A new pipeline handler lets users configure up to 4 binary sensors, each paired with a fixed cover position. When a sensor is "on", the cover moves to that position. Sensors are evaluated in priority order (1→4); the first active sensor wins. If all sensors are off, the handler passes through silently.

**Use cases:**
- Scene-based positioning (e.g., "Movie Mode" sensor → close all blinds to 5%)
- Schedule-driven automation (e.g., "Morning Scene" sensor → set blinds to 50%)
- Automation overrides (pair with Home Assistant automations for custom logic)

**Priority:** 77 (sits between manual override at 80 and motion timeout at 75)
- Manual moves still take precedence (user always wins)
- Custom positions override automatic sun tracking and climate modes
- Motion timeout fallback still works when no custom sensors are active

**Config Flow:**
- New "Custom Positions" step in both setup and options flows
- 4 independent sensor/position pairs
- Positioned in the config menu right before Motion Override
- Full translations in 13 languages (English + placeholders)

## 🔧 Technical Details

- New `CustomPositionHandler` at priority 77
- `PipelineSnapshot.custom_position_sensors` carries ordered (entity_id, is_on, position) tuples
- Coordinator syncs binary sensor states each update cycle
- Config validation ensures sensor and position are paired correctly

## 🐛 Known Limitations

- Sensors currently checked only on state changes; no polling fallback
- No visual indicator in config flow showing if sensors are currently "on"
- Position values must be 0-100%; no custom limits per sensor yet
