## 🎯 Config UX Simplification

Major configuration UX improvements to reduce complexity — **zero runtime behavior changes** for existing users.

### ✨ Features

- **Quick Setup flow** — New menu after cover type selection offers Quick (5 screens) vs Full (all screens). Quick skips overrides, climate, and automation — perfect for first-time setup. All settings remain accessible via the Options menu.
- **Split Climate screen** — The overloaded Climate screen is now two focused screens:
  - ☁️ **Light Sensors & Cloud Suppression** — weather entity, lux, irradiance, cloud coverage, and suppression toggle all in one place
  - 🌡️ **Temperature & Climate Mode** — indoor/outdoor temp, presence, winter insulation
- **Weather Conditions merged** — Sunny state selector folded into the Light & Cloud screen, eliminating a standalone step
- **Position Map in summary** — New section showing at a glance what position applies in each scenario (safety override, weather danger, sun tracking, sunset, default)
- **Options menu grouping** — Emoji prefixes visually organize the menu: 📐 Setup, 📊 Position, ⏱️ Schedule, ☁️🌡️ Environment, 🛡️ Overrides, ⚙️ Admin
- **Sync feature promoted** — Moved higher in options menu with a clearer label for multi-window management

### 🔧 Improvements

- **Reduced switch sprawl** — Lux, Irradiance, and Outside Temperature switches now hidden by default for new installations (`entity_registry_enabled_default=False`). Existing installations are unaffected — entity registry preserves current state.

### ⚠️ No Breaking Changes

- All modifications are UI/config flow only
- Existing cover positions, pipeline decisions, and sensor readings are unchanged
- Existing switches remain visible (entity registry preserves enabled state)
- Cloud suppression still requires explicit toggle (no auto-enable)
- Legacy combined Climate screen remains available in options flow for backward compatibility
