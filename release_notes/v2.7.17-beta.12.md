## 🐛 Fix Decision Trace Sensor Startup Error

### 🐛 Bug Fixes
- Fix `decision_trace` sensor failing to load on startup — newer Home Assistant versions raise a `ValueError` for sensors with an empty-string unit (`""`) returning non-numeric values. Switched sensor to `SensorDeviceClass.ENUM` with explicit options matching all `ControlMethod` values (`solar`, `default`, `manual_override`, `motion_timeout`, `force_override`, `wind_override`, `summer`, `winter`, `cloud_suppression`).

### 🧪 Testing
- All 751 unit tests passing
- Tested against Home Assistant 2024.5.0+
