## 🎯 Beta — Surface manual-override gate state in logs and diagnostics

Beta.1 adds debug logging and diagnostics visibility for the manual override detection gate, making it easier to diagnose cases where manual override isn't being detected.

### ✨ Features

- **Manual override gate logging (#280):** When a cover service call or state change is silently skipped because Manual Override Detection or Automatic Control is off, a DEBUG log is now emitted. Previously these were silent returns, making it impossible to tell from logs why manual override wasn't activating.
- **Manual override gate in diagnostics (#280):** `manual_toggle` and `enabled_toggle` (Automatic Control) booleans are now included in the diagnostics JSON under `configuration`, so the state of these gates is visible in every diagnostics dump.
- **Rename switch to "Manual Override Detection" (#280):** The Manual Override switch display name is now "Manual Override Detection" to better reflect its function. The entity `unique_id` is unchanged — no registry entries are affected.
