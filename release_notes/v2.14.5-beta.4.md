## 🐛 False Manual Override After Cancel — Startup Delay Fix

### 🐛 Bug Fixes

- **False manual override on covers with hardware startup delay**: Fixed a regression where canceling a manual override and then letting the integration move a cover would immediately re-trigger a manual override. Covers with a hardware startup delay (10–30+ seconds between receiving a command and physically moving) fire their first state event before the motor engages. The strict distance comparison (`new < old`) treated equal distances as "not in transit" and cleared `wait_for_target`, causing the next position event to appear as a user move. The fix distinguishes a startup delay (state just transitioned from `closed` → `opening`, position unchanged) from a genuine mid-transit stall (already `opening`, position unchanged) — the former now correctly keeps `wait_for_target=True`.
