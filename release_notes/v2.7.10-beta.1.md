## 🎯 Solar Time Clamping — Beta 1

### ✨ Features

- **Sun Time Clamping** — 6 new optional automation settings to control when sun tracking starts and ends each day:
  - **Minimum Start Time** — Delay tracking if the sun enters your window's FOV too early (e.g. 5 AM in summer)
  - **Maximum Start Time** — Force tracking to begin by a specific time even if the sun enters FOV later
  - **Minimum End Time** — Keep tracking active until at least this time even if the sun exits FOV early
  - **Maximum End Time** — Always stop tracking by this time regardless of when the sun leaves FOV
  - **Start Elevation Threshold** — Start tracking only when the sun rises above a set angle (e.g. 8° prevents very low morning glare)
  - **End Elevation Threshold** — Stop tracking when the sun drops below a set angle (e.g. 5° stops before full sunset)

### 🔧 Technical Details

- All 6 options are fully optional with no defaults — existing configurations are unaffected
- `"00:00:00"` is treated as "disabled", consistent with the existing Start/End Time convention
- When min/max time and elevation threshold are both set for the same boundary, the more restrictive value is used
- An inversion guard reverts to raw solar times (with a warning) if clamping would produce start > end
- Clamping is applied after the underlying `solar_times()` calculation — the core algorithm is unchanged

### 🧪 Testing

- 19 new unit tests covering all 6 settings, combined scenarios, edge cases, and the inversion guard

### 📦 Installation

Update via HACS or replace `custom_components/adaptive_cover_pro/` manually.

After updating, reconfigure any cover instance → **Automation** step to see the new Sun Time Clamping fields (all optional, leave empty to disable).
