## 🎯 Optional Device Association

### ✨ Features

- **Device Association** — Integration entities (sensors, switches, buttons) can now be merged directly into an existing physical device (e.g., a blind motor or cover controller) instead of appearing as a standalone virtual "Adaptive Cover" service device.
  - During initial setup, a new **"Link to Existing Device"** step appears when the selected cover entities belong to known HA devices. Only devices that own the configured cover entities are shown.
  - The options flow now includes a permanent **"Device Association"** menu item so you can attach or remove the device link at any time without reconfiguring.
  - Selecting **"None (standalone device)"** at any time reverts to the previous behavior — a separate virtual Adaptive Cover device.
  - Falls back gracefully to standalone mode if the linked physical device is later removed from HA.

### 🧪 Testing

- 11 new unit tests covering helper logic, device_info merging/fallback, config flow storage, and options flow set/removal
- 324 total tests passing
- Tested with Python 3.11+ and Home Assistant 2024.5.0+

### 📦 Installation

Update via HACS or download the ZIP from this release and copy `custom_components/adaptive_cover_pro` into your HA config directory. Restart Home Assistant.

> **Beta note:** Device association is a new UI flow. If you encounter any issues, open an issue on GitHub. The feature is fully optional — existing configurations are unaffected.
