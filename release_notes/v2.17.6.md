## 🐛 Bug Fix — End-time safety target causes cover to reopen after manual override (#215 #216)

### 🐛 Bug Fixes

- **Cover reopens at night after manual override expires (#215):** The end-time transition (`_on_window_closed`) was sending the end-time default with `force=True`, which tagged the target as a "safety target" in the cover command service. Safety targets bypass the time-window gate in reconciliation. When a manual override expired hours later (outside the automation window), the reconciler saw the stale 100% safety target, bypassed the outside-window skip, and sent `set_cover_position=100` — reopening the cover at night with sunset active. Fixed by changing `_on_window_closed` to use `force=False` so the target is never safety-tagged.

- **Safety target lingers past manual override (#216):** As a belt-and-suspenders companion fix, a new `discard_target()` method on `CoverCommandService` is called whenever a cover enters manual override. This immediately clears any pre-existing integration target (including any safety-tagged targets) so reconciliation cannot resurrect it while or after the user is in control.
