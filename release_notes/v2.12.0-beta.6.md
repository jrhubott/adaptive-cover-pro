# v2.12.0-beta.6 — Climate Mode Irradiance Fix

## 🎯 Summary

Fixed Issue #71: Irradiance sensors (pyranometers) and lux sensors are now properly respected in summer conditions. When irradiance drops below the configured threshold, covers will use the default (open) position rather than closing based on solar geometry, even during summer months.

## 🐛 Bug Fixes

- **Issue #71** — Irradiance/lux/weather signals now suppress glare control in summer mode
  - Previously: Irradiance was ignored when summer temperature threshold was exceeded
  - Now: Low-light conditions (lux, irradiance, or non-sunny weather) trigger LOW_LIGHT strategy in all seasons
  - Example: If outdoor temp is 30°C (summer) but irradiance sensor reads 200 W/m² (below 500 W/m² threshold), covers stay open instead of closing

## 📝 Technical Details

**Modified:** `pipeline/handlers/climate.py`
- `normal_with_presence()` — Low-light check moved before season logic
- `normal_without_presence()` — Low-light check overrides summer cooling
- `tilt_with_presence()` — Restructured with elif chain; low-light before summer
- `tilt_without_presence()` — Low-light check added before summer cooling

**Preserved:** Winter HEATING still takes priority over low-light (if cold, open for solar gain)

**Tests:** 10 regression tests added (972 total passing)
