# Adaptive Cover Pro v2.13.6

## 🎯 Major Updates: Pipeline Refactor, Motion Control, and Manual Override Fixes

This stable release consolidates significant improvements to the pipeline architecture, adds motion-based automatic positioning control, and fixes critical bugs in manual override behavior.

---

## ✨ Features

### **Pipeline Time Window Gating** (v2.13.6-beta.1)
- **Solar and Glare Zone handlers now self-gate** on the configured time window (start/end time)
- When outside the time window, these handlers skip evaluation and pass through to lower-priority handlers
- Cleaner decision traces — "outside time window" clearly shown when handlers are gated
- Removed redundant time-window gate from `CoverCommandService` — responsibility now owned by pipeline handlers
- Result: More transparent behavior and easier debugging

### **Motion Control Switch** (v2.13.6-beta.1)
- **New optional toggle switch** to enable/disable motion-based automatic positioning
- When disabled, motion timeout is bypassed — covers controlled by sun position, climate, or default
- Conditionally shown only when motion sensors are configured (backward compatible)
- Allows users to temporarily override motion timeout (e.g., during cleaning or maintenance)

### **Improved Configuration Summary UI** (v2.13.6 stable)
- **Timing lines now properly indented** in config flow summary using non-breaking spaces
- **End-time and sunset positions show separately** when they differ
  - `🔚 After end time → 60%` (position when scheduled window ends)
  - `🌅 After sunset → 0%` (position after astronomical sunset)
- Clearer distinction for users with distinct timing and sunset behaviors

## 🐛 Bug Fixes

### **Manual Override Reset Position Bug** (v2.13.6-beta.3)
- **Fixed:** Covers were not returning to the correct pipeline-calculated position after manual override expired or was reset
- **Root cause:** Reset flow was not refreshing the pipeline before sending the position
- **Solution:** Reset button now correctly:
  1. Clears the manual override
  2. Runs the full pipeline calculation
  3. Sends the authoritative post-override position (climate-aware if climate mode active)
- **Impact:** Covers now return to their correct position immediately upon reset, instead of lingering at the manually-moved position

### **Reset Button Time Delta Gate Regression** (v2.13.6-beta.4)
- **Fixed:** Reset button was blocked by `time_delta_too_small` gate when pressed within 2 minutes of the manual move
- **Root cause:** Gate checked `entity.last_updated` (set by any cover state change, including user moves)
- **Solution:** Reset button now uses `force=True` to bypass all gates (delta, time threshold, auto_control)
- **Rationale:** User-initiated reset is an explicit request for pipeline position — gates should not block it

## 🔧 Technical Changes

### Pipeline Architecture
- `SolarHandler` and `GlareZoneHandler` now return `None` when outside time window (self-gating)
- `MotionTimeoutHandler` respects the new `motion_control_enabled` snapshot field
- `PipelineSnapshot` now includes `in_time_window` and `motion_control_enabled` gates
- `CoverCommandService` no longer maintains `in_time_window` context (moved to handlers)
- Manual override reset flow moved to `AdaptiveCoverManager.reset_if_needed()` — runs pre-pipeline

- ✅ All scenarios tested: time window gating, motion control toggle, climate-aware reset
