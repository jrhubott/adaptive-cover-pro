## 🐛 Fix: Diagnostic sensors failing to register on startup

### 🐛 Bug Fixes

- **Fix diagnostic sensors dropped from registry on startup (#203)** — The v2.17.2 fix was incomplete. Diagnostic sensors (`sun_position`, `control_status`, `climate_status`, `decision_trace`, and others) still crashed with `AttributeError: 'NoneType' object has no attribute 'diagnostics'` when the entity-add task raced `first_refresh`. Affected sensors were silently dropped from the HA registry for the whole session and never came back without a full reload. The per-class availability guards added by the previous fix are replaced with a single guard in `AdaptiveCoverBaseEntity` so every current and future entity in every platform inherits it automatically.

### 🧪 Testing

- Added `tests/test_sensor_availability.py` with 19 tests covering all 18 concrete entity classes, including explicit regression tests that reproduce the `sensor.py:353` and `sensor.py:471` crashes from the bug report, and a reflection-based completeness test that fails when a new entity class is added without availability coverage.
- **2024 tests passing.**
