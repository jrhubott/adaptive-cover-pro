## 🎯 Cloudy Position, Summer Presence Fix, and Presence Entity Selector Restoration

### 🔑 Highlights

- New `cloudy_position` option lets you set a distinct cover position for cloudy/weather-suppressed daytime — separate from the generic default position (#311).
- Summer mode no longer closes blinds that aren't in the sun's path when presence is active (#314).
- Presence entity selectors now support all five original domains (device_tracker, person, zone, binary_sensor, input_boolean) after a regression in v2.18.4 narrowed them to two (#313).

### ✨ Features

- **Separate cloudy-day position — `cloudy_position` option (#311):**
  Previously, both "sun outside the window's azimuth/elevation range" and "sun in the window but weather suppression is active" fell back to the same `default_position`, making it impossible to hold covers at different heights for the two scenarios. The new optional `cloudy_position` field (0–100%, slider) is returned by the cloud-suppression handler instead of `default_position` when cloud/weather/lux/irradiance suppression fires. The sunset window still takes priority when active. A ⚠️ warning appears in the config summary when `cloudy_position` is set but cloud suppression is disabled.

### 🐛 Bug Fixes

- **Summer mode no longer closes shaded blinds when presence is active (#314):**
  `normal_with_presence()` was closing transparent blinds to 0% in summer even when the sun was not hitting the window (`cover.valid` was `False`). The fix adds the `and self.cover.valid` guard to the summer+transparent+presence path, making it symmetric with `normal_without_presence()`. Blinds shaded from the sun now defer to the glare/solar handlers instead of snapping closed.

- **Presence entity selector restored to five domains (#313):**
  A regression introduced in v2.18.4 (#287) silently narrowed the presence entity selector from its original five domains — `device_tracker`, `person`, `zone`, `binary_sensor`, `input_boolean` — down to just `binary_sensor` and `input_boolean`. Users relying on `person`, `zone`, or `device_tracker` entities for presence control could no longer select them in the config UI. The original five-domain set is restored, and the presence reader now treats `person` entity home/not_home states the same way it handles `device_tracker`.
