# Adaptive Cover Pro v2.13.6-beta.2

## 🎯 Pipeline Refactor: Time Window & Motion Control Gating

This beta release moves the time window gate into the pipeline handlers and adds a new **Motion Control** switch to enable/disable motion-based automatic positioning.

### ✨ Features

- **Time Window Gating in Pipeline** — `SolarHandler` and `GlareZoneHandler` now self-gate on the time window, providing cleaner decision traces. When outside the configured start/end time, these handlers skip and pass through to lower-priority handlers.
- **Motion Control Switch** — New toggle switch to enable/disable motion-based automatic positioning. When disabled, motion timeout is bypassed, and sun/climate/default handlers take over. Conditionally shown only when motion sensors are configured.
- **Pipeline Snapshot Gates** — New `in_time_window` and `motion_control_enabled` fields on `PipelineSnapshot` allow any handler to gate its logic; currently used by Solar, Glare Zone, and Motion Timeout handlers.

### 🔧 Technical Changes

- `SolarHandler` and `GlareZoneHandler` return `None` when outside time window
- `MotionTimeoutHandler` returns `None` when Motion Control switch is off
- Removed `in_time_window` gate from `CoverCommandService.apply_position()` — time-window responsibility moved to pipeline handlers
- `PositionContext` no longer has `in_time_window` field (context.auto_control gate remains)
- `ToggleManager.motion_control` tracks switch state; coordinator delegates to it
- All tests updated; 1090 tests passing

### 🧪 Testing

- ✅ 7 new tests for time window and motion control gating
- ✅ Backward compatible — both new snapshot fields default to `True`
- ✅ Motion Control switch shown only when motion sensors configured
- ✅ Full pipeline integration tested with all gating scenarios

3. Time Window gating — check Decision Trace sensor; Solar/Glare handlers skip with "outside time window" when outside window
