## 🎯 Full Architecture Rewrite

Major internal rewrite of the integration architecture. Zero behavior change — all existing configurations and automations continue to work without modification.

### ✨ Architecture Improvements

- **Pure Calculation Engine** — `calculation.py` and `sun.py` have zero Home Assistant imports. Cover position math is fully unit-testable without HA mocks.
- **State Providers** — New `state/` package with `ClimateProvider` and `SunProvider`. All HA entity reads happen at the boundary layer, producing pure data for the calculation engine.
- **Override Pipeline** — New `pipeline/` package with 6 pluggable handlers replacing scattered if/elif chains. Adding a new override type (wind, cloud suppression) requires creating one handler file.
  - Force Override (100) > Motion Timeout (80) > Manual Override (70) > Climate (50) > Solar (40) > Default (0)
- **Manager Classes** — 5 focused managers extracted from the coordinator: `ManualOverrideManager`, `GracePeriodManager`, `MotionManager`, `PositionVerificationManager`, `CoverCommandService`
- **Diagnostics Builder** — Extracted diagnostic data construction with decision trace from pipeline evaluation
- **Coordinator Slimmed** — 2,700 → 1,839 lines (-32%), now a thin orchestrator

### 🧪 Testing

- **657 tests** (was 442), **58% coverage** (was 38%)
- Calculation tests no longer require Home Assistant mocks
- Each new module has dedicated test coverage
- Tested with Python 3.11 and 3.12
- Home Assistant 2024.5.0+
