## 🎯 Fix False Manual Override on Slow Covers

### 🐛 Bug Fixes

- **Slow covers no longer trigger false manual override (#271):** Covers that take more
  than 45 seconds to fully close or open were incorrectly being marked as manually
  overridden. The transit backstop was clearing `wait_for_target` based on
  time-since-command-sent rather than time-since-last-progress, so a cover still
  actively moving would appear stalled and the next intermediate position report would
  exceed the manual override threshold. The backstop now resets each time the cover
  reports forward progress toward the target. Stalled covers and those that report no
  intermediate positions fall back to the original sent_at measurement. Both the
  coordinator and reconciliation backstops share the same logic.

- **Configurable transit timeout (#271):** A new **Transit Timeout** setting in
  Debug & Diagnostics (15–600 s, default 45 s) lets users with large or slow covers
  extend the wait window without code changes.
