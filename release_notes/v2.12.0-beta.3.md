## 🌦️ Weather Override — Beta 3

This release adds comprehensive weather-based safety override support, addressing Issue #28 (wind speed/direction handling) and extending it to cover rain, severe weather, and other weather conditions.

### ✨ Features

- **Weather Override** — New `WeatherManager` evaluates multiple weather conditions and retracts covers when any threshold is exceeded:
  - **Wind speed sensor** — configurable threshold (km/h or m/s); optional wind direction filter to only trigger when wind is aimed at the window (within a configurable tolerance)
  - **Rain rate sensor** — configurable threshold (mm/h)
  - **Is-raining binary sensor** — immediate trigger when "on"
  - **Is-windy binary sensor** — immediate trigger when "on"
  - **Severe weather binary sensors** (hail, frost, storm, etc.) — any "on" triggers override
- **Clear-delay timeout** — Configurable delay (seconds) after conditions clear before covers resume normal control, preventing flapping during gusty or intermittent weather
- **Weather pipeline handler** — New `WeatherHandler` at priority 85 (between wind/90 and motion/80) integrates weather override into the existing pipeline architecture; old wind stub handler removed
- **OR logic** — Any single active condition triggers the override; all conditions must clear before the delay countdown begins
- **Fail-open** — Unavailable or unknown sensor states are treated as inactive (covers are not retracted on sensor failure)

### 🐛 Bug Fixes / Changes

- Removed the wind stub handler (`wind.py` at priority 90) — replaced by the fully-implemented `WeatherHandler`
