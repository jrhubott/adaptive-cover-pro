## 🎯 Custom Position Reliability, Wider Window Depth Range, and Slow Cover Fix

### ✨ Features

- **Custom position slots now bypass delta gates and auto-control (#275):** When a
  custom position sensor fires, the cover command is always sent immediately — the
  global Minimum Delta Time and Minimum Delta Position throttles no longer block the
  move. Custom positions also bypass the automatic control gate, matching the existing
  behavior of force-override and weather handlers.

- **Window depth maximum raised to 5.0 m (#276):** Users with deep architectural
  reveals — recessed windows, thick masonry, or balcony overhangs — can now enter
  realistic values. The underlying geometry scales naturally; only the UI slider,
  options-service validator, and service description needed updating.

### 🐛 Bug Fixes

- **Slow covers no longer trigger false manual override (#271):** Covers that take
  more than 45 seconds to fully close or open were incorrectly being marked as
  manually overridden. The transit backstop was clearing `wait_for_target` based on
  time-since-command-sent rather than time-since-last-progress, so a cover still
  actively moving would appear stalled and the next intermediate position report would
  exceed the manual override threshold. The backstop now resets each time the cover
  reports forward progress toward the target. Stalled covers and those that report no
  intermediate positions fall back to the original sent_at measurement. Both the
  coordinator and reconciliation backstops share the same logic.

- **Configurable transit timeout (#271):** A new **Transit Timeout** setting in
  Debug & Diagnostics (15–600 s, default 45 s) lets users with large or slow covers
  extend the wait window without code changes.
