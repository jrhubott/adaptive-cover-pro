## ⚠️ Beta Release v2.17.8-beta.4

This is a pre-release for testing. Please report any issues before the stable release.

## 🐛 Bug Fixes

### Fix sun entity time window rollover (Refs #226)
When `sensor.sun_next_rising` or `sensor.sun_next_setting` is used as a start or end time entity, the sensor rolls forward to tomorrow's datetime once the event passes. This caused `after_start_time` to return `False` for the rest of the day, silently disabling cover tracking. The fix normalises any future-dated entity time back to today's date so time window comparisons remain correct throughout the day.

### Fix glare zone priority — zones now override climate's solar tracking (Refs #231)
When climate mode was enabled, the glare zone handler (priority 45) was always outprioritized by the climate handler (priority 50), making glare zones completely ineffective. The fix raises glare zone priority to 55 (above climate, below cloud suppression).

### Fix cloud suppression with unavailable weather entity (Refs #222)
When a weather entity becomes unavailable or is removed, cloud suppression was treating the unavailable state as "not sunny" and incorrectly staying active. An unavailable weather entity is now treated as sunny (skip check).

### Fix per-user language in options menu (Refs #227)
The options flow menu was rendering labels in the HAOS system language instead of each user's profile language. Menu options are now passed as step IDs and translated client-side by the HA frontend.
