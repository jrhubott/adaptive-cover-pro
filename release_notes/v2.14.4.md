## 🐛 Bug Fix: Force Override Release Now Immediately Repositions Cover

### 🐛 Bug Fixes

- **Force override release respects time delta (#177):** When a force override sensor deactivates (e.g. a window closes), the cover now immediately returns to its calculated position. Previously, the time delta gate blocked this move because it counted the force override's own position change as the "last move." The coordinator now detects the on→off transition and bypasses time and position delta gates, matching the behavior already used for manual override expiry.

### 🧪 Testing

- Added 10 new tests (1767 total): 5 coordinator-level regression tests covering the force override release transition, and 5 end-to-end gate tests in `CoverCommandService` documenting the pre-fix failure mode and confirming `force=True` bypasses both time delta and position delta gates.
