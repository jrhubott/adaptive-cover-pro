## 🐛 Bug Fix + ✨ Config Flow Improvement

### 🐛 Bug Fixes

- **Force override release no longer blocked by time delta (#177):** When a force override sensor deactivates (e.g. a window closes), the cover now immediately returns to its calculated position. Previously, the time delta gate blocked the repositioning because it counted the force override's own position change as the "last move" — causing `last_skipped_action` to show `time_delta_too_small` until the threshold elapsed. The coordinator now tracks force override state across update cycles and detects the on→off transition, bypassing time and position delta gates (matching the existing behavior for manual override expiry). The fix also ensures repositioning occurs even if the release happens outside the configured active-hours window.

### ✨ Improvements

- **Config flow menu icons embedded in code:** Menu icons for the options flow are now defined in `config_flow.py` (the `MENU_ICONS` dict) and applied at runtime, rather than being encoded in each translation file. Menu labels are loaded from the active HA language at display time via `async_get_translations`, so all languages show consistent icons alongside their translated text. This simplifies future translation updates — icons no longer need to be duplicated across 13 language files.

### 🧪 Testing

- Added 10 new tests (1767 total): 5 coordinator-level regression tests covering the force override on→off transition (force=True passed on release, correct reason string, time-window edge case), and 5 end-to-end `CoverCommandService` gate tests that document the pre-fix failure mode and verify `force=True` bypasses both time and position delta gates.
- Removed unused imports across multiple test files (test suite cleanup).
