## 🎯 Window Geometry Fields Now Support Up to 50 m

Beta.2 adds the geometry field limit increase from beta.1 (custom position redundant-command fix) and raises the cap on four window-geometry fields from their previous 3–6 m limits to 50 m, switching them to numeric box input. Also includes the custom position redundant-command fix from beta.1.

### 🔑 Highlights

- Window height, width, sill height, and distance-to-shaded-area can now be configured up to 50 m — supporting sports halls, atria, and deep industrial spaces (#301 #302).
- Custom position covers no longer send redundant positioning commands every few seconds (#290).

### ✨ Features

- **Geometry field limits raised to 50 m (#301 #302):**
  The previous caps (3–6 m depending on field) made accurate geometry configuration
  impossible for large spaces. All four fields — window height, window width, sill
  height, and distance to shaded area — now accept values up to 50 m. Input style
  switched from slider to numeric box to make entering large values practical.

### 🐛 Bug Fixes

- **Custom position covers send redundant commands on every sun update (#290):**
  Regression from v2.18.3 where `CustomPositionHandler` gained `bypass_auto_control=True`,
  which was incorrectly equated with "safety handler", causing `force=True` on every
  update and bypassing the same-position short-circuit. Fixed by separating
  `_pipeline_is_safety_handler` (force override, weather only) from
  `_pipeline_bypasses_auto_control`. Defence-in-depth: same-position guard lifted
  to top-level in `apply_position` to cover all future force=True paths.
