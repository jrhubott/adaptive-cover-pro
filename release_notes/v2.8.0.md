## 🏗️ Architecture Rewrite Release

v2.8.0 is a major internal rewrite of Adaptive Cover Pro. The integration's core has been restructured into a clean layered architecture — improving maintainability, testability, and extensibility — while preserving full backward compatibility for existing installations.

This release consolidates 12 beta releases (v2.7.17-beta.1 through beta.12) into a single stable version.

---

### 🏛️ Architecture Overhaul

The coordinator has been decomposed from a 2,700-line monolith into focused, single-responsibility modules:

- **Pure Calculation Engine** — `calculation.py` and `sun.py` now have zero Home Assistant imports, making them fully unit-testable without HA mocks
- **State Providers** (`state/`) — All Home Assistant entity reads happen at the boundary layer via `ClimateProvider`, `SunProvider`, and `CoverProvider`
- **Override Pipeline** (`pipeline/`) — 8 pluggable handlers replace scattered if/elif chains with a clean priority system: Force Override (100) > Wind (95, stub) > Motion Timeout (80) > Manual Override (70) > Climate (50) > Solar (40) > Cloud Suppression (35, stub) > Default (0)
- **Manager Classes** (`managers/`) — 5 focused managers extracted from the coordinator: `ManualOverrideManager`, `GracePeriodManager`, `MotionManager`, `PositionVerificationManager`, `CoverCommandService`
- **Diagnostics Builder** (`diagnostics/`) — Decision trace construction extracted from the coordinator
- **Typed Configuration** — `CoverConfig`, `VerticalConfig`, `HorizontalConfig`, `TiltConfig` frozen dataclasses replace 22 fragile positional arguments
- **SunGeometry Engine** (`engine/`) — Pure sun angle math extracted into its own module
- **Coordinator** — Reduced from ~2,700 to 1,477 lines (-45%), now a thin orchestrator

---

### ✨ New Features

- **Decision Trace Sensor** — New diagnostic sensor showing the full pipeline evaluation chain. State is the winning handler name; attributes include the complete trace with each handler's decision and reason
- **Enriched Cover Position Sensor** — Now includes `control_method`, `reason`, `position_explanation`, and `raw_calculated_position` as attributes
- **Position Explanation** — Human-readable decision chain (e.g. `"Sun tracking (45%) → Climate: Winter Heating → 100%"`) available as a sensor attribute and recorded in HA history
- **Venetian Cover Calculation Engine** — `VenetianCoverCalculation` with `DualAxisResult` for dual-axis covers controlling both position and tilt (Issue #33). Calculation engine is complete; config UI coming in a future release
- **Wind & Cloud Override Stubs** — Pipeline handlers for wind override (Issue #28) and cloud suppression (Issue #31) are registered and ready for implementation
- **Duplicate & Sync Covers** — "Duplicate existing cover" option in Add Integration flow; "Sync settings to other covers" option in every cover's Options flow (Issue #54)
- **Simulation Notebook** — `notebooks/simulate_cover.ipynb` for visualizing cover behavior for any day or date range without running Home Assistant
- **Comprehensive Debug Logging** — Enable `custom_components.adaptive_cover_pro: debug` in your logger config for detailed logs at every decision point: position changes, skipped moves, entity state changes, sun tracking transitions, and full geometric calculation details
- **Last Skipped Action Sensor** — Diagnostic sensor recording why the most recent automatic cover move was suppressed (time window, position delta too small, time delta too small, manual override active)
- **Calculation Detail Attributes** — Five new attributes on Calculated Position for vertical covers: `edge_case_detected`, `safety_margin`, `effective_distance`, `window_depth_contribution`, `sill_height_offset`

### ⚠️ Breaking Changes: Diagnostic Sensor Consolidation

The diagnostic sensor system has been streamlined:

- **Removed** the "Enable diagnostic entities" config toggle — all devices now always get diagnostic sensors
- **Consolidated** from 17 sensors down to 9 cleaner, richer sensors:
  - **Sun Position** — azimuth (state) + elevation, gamma, FOV bounds, in_fov as attributes
  - **Control Status** — combines Control Status + Control State Reason + Time Window Status + Sun Validity Status, with debug timing attributes
  - **Calculated Position** — enriched with `sun_calculated_position` attribute
  - **Position Verification** — combines verification status + retry count
  - **Motion Status** — meaningful states: "Motion Detected", "Timeout Pending", "No Motion"
  - **Climate Status** — combines temperature + climate conditions (when climate mode enabled)
  - **Decision Trace** — full pipeline evaluation chain
  - **Last Skipped Action** — most recent suppressed move and reason
  - **Position Explanation** — human-readable decision chain
- **Removed** 3 standalone sensors: "Control Method", "Calculated Position", and "Position Explanation" sensors — their data is now available as attributes on the Cover Position sensor

**Upgrade Note:** Old entity registry entries are automatically cleaned up on restart. If you have automations or dashboard cards referencing old entity IDs (e.g., `sensor.my_cover_sun_azimuth`, `sensor.my_cover_sun_elevation`), update them to the new consolidated sensors (e.g., `sensor.my_cover_sun_position` with elevation as an attribute).
