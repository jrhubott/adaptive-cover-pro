## 🐛 Bug Fix: Coordinator Crash on Every Update Cycle

### 🐛 Bug Fixes

- **Fix coordinator AttributeError on every update cycle** — `build_diagnostic_data()` referenced `self.last_update_success_time` which does not exist on Home Assistant's `DataUpdateCoordinator` base class. This caused all cover instances to fail their update cycle continuously (introduced in v2.15.1-beta.1). Tracked as own attribute `_last_update_success_time`.

### 🧪 Testing

- 1892 tests passing
- Added regression test to ensure `_last_update_success_time` attribute exists on real coordinator instances
