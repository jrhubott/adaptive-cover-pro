## 🎯 Cloudy position — separate cover position for cloud-suppressed state

### 🔑 Highlights

- New optional `cloudy_position` config option (0–100%) in the Light & Cloud step. When set, the cloud-suppression handler moves covers to this position instead of the generic default — letting you distinguish "cloudy daytime" from "sun outside the window."

### ✨ Features

- **Cloudy position (#311):** Adds a `cloudy_position` slider (0–100%) to the Light & Cloud configuration step. When cloud suppression fires (weather not sunny, lux below threshold, irradiance below threshold, or cloud coverage above threshold), covers move to this position instead of `default_position`. The sunset window still takes priority when active. Position limits (min/max) apply to `cloudy_position` the same as any other handler output.
  - Diagnostic data surfaces in three places: `configuration.cloudy_position` + `configuration.cloud_suppression_enabled` in the config block, `default_position.configured_cloudy_pos` in the position block, and the decision trace reason string automatically.
  - The config summary shows a ⚠️ warning when `cloudy_position` is set but cloud suppression is disabled — nothing to suppress, so the value has no effect.
  - Translations: en/de/fr synced.
