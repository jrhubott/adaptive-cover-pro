## ЁЯОп Diagnostics Overhaul, Lovelace Card Service, and Auto-Control Gate Fixes

### ЁЯФС Highlights

- New `get_diagnostics` service returns a live diagnostics snapshot for the companion Lovelace card.
- Covers are no longer commanded when Automatic Control is switched off тАФ `force=True` no longer bypasses the auto-control gate (#293).
- False manual override detection is fixed for covers that report `open` state throughout transit (#285).
- The event buffer is expanded to 250 entries (max 1000) with 12 new lifecycle event types for richer diagnostics.
- `input_boolean` entities are now accepted in all boolean-gate selectors (force override, motion, custom position, weather, presence) (#284).

### тЬи Features

- **`get_diagnostics` service for Lovelace card integration:** A new response-capable service (`adaptive_cover_pro.get_diagnostics`) returns the live diagnostics snapshot тАФ pipeline trace, cover positions, climate readings, sun data, coordinator health тАФ for any targeted ACP instance. Designed to be called from the companion Lovelace card via `hass.callService` with `return_response: true`. The `_sanitize()` helper is also hardened to handle numpy scalars, Enum values, dataclasses, and sets, improving reliability of both this service and the existing HA diagnostics download.

- **Expanded event buffer coverage and increased capacity:** The event buffer default is raised from 50тЖТ250 entries and the maximum from 200тЖТ1000, to support richer transit and lifecycle diagnostics without data loss. Twelve new event recording call sites are added across the coordinator and managers, covering: `transit_progress_forward`, `transit_timeout_cleared`, `transit_startup_delay`, `transit_cleared`, `manual_override_gate_closed`, `sun_entered_fov`, `sun_left_fov`, `end_time_default_sent`, `sunset_window_opened`, `grace_period_expired`, `startup_grace_expired`, and `motion_timeout_started/expired/canceled/detected_during_timeout`.

- **Event timeline in diagnostics (#294/#295):** The diagnostics JSON now includes an `event_timeline` ring buffer capturing up to 250 event types across the full cover lifecycle тАФ cover commands, pipeline evaluations, manual override transitions, `wait_for_target` lifecycle, and override state changes. `last_cover_action` gains 11 new fields including `target_source` (which caller sent the command), force/safety flags, integration state at call time, pipeline decision trace, and a `gates_evaluated` shadow check showing which gates `force=True` bypassed. The existing `manual_override_history` key is retained as a backward-compatible filtered view.

- **`enabled_handlers` attribute on decision_trace sensor (#297):** The `decision_trace` sensor now exposes an `enabled_handlers` list attribute naming the pipeline handlers that are actually configured for this cover instance. The companion Lovelace card uses this list to hide unconfigured handlers (e.g. a Force Override row when no force-override sensors are set). Configuration-based, not runtime-based: a handler appears in the list if its options are populated, regardless of whether its runtime switch is currently on.

- **`input_boolean` accepted in all boolean-gate selectors (#284):** Force override sensors, custom position sensors, motion sensors, weather sensors (rain, wind, severe), and the presence entity selector now accept `input_boolean` entities alongside `binary_sensor`. The presence entity selector is also narrowed from four domains (including `device_tracker` and `zone`) to two, since device_tracker and zone states are never `on`/`off`.

- **Manual override gate logging and diagnostics (#280):** When a cover service call or state change is silently skipped because Manual Override Detection or Automatic Control is off, a DEBUG log is now emitted. Previously these were silent returns, making it impossible to tell from logs why manual override wasn't activating. `manual_toggle` and `enabled_toggle` (Automatic Control) booleans are also included in the diagnostics JSON under `configuration`.

- **Rename switch to "Manual Override Detection" (#280):** The Manual Override switch display name is now "Manual Override Detection" to better reflect its function. The entity `unique_id` is unchanged тАФ no registry entries are affected.

### ЁЯРЫ Bug Fixes

- **Auto-control gate bypassed by force=True (#293):** When Automatic Control was switched off, the integration could still issue cover commands on certain internal paths that passed `force=True`. A cover commanded while the user was manually repositioning it would overwrite the user's position and leave `wait_for_target=True`, blocking subsequent manual moves. The fix ensures `apply_position` always respects the auto-control gate, and introduces `PositionContext.bypass_auto_control` as the sole sanctioned escape hatch (used only by the switch return-to-default one-shot). `async_handle_cover_state_change` also now correctly observes manual moves even when `auto_control=False`.

- **EventBuffer crashes on startup when buffer size is a float:** HA's `NumberSelector(SLIDER)` returns floats (e.g. `50.0`) for `CONF_DEBUG_EVENT_BUFFER_SIZE`. Passing a float to `deque(maxlen=...)` raises `TypeError`, preventing integration setup. The fix coerces at the `EventBuffer` boundary so any caller is safe.

- **False manual override on covers that stay "open" during transit (#285):** The transit progress check in `process_entity_state_change()` was gated on the cover emitting `opening` or `closing` HA state. Covers that stay `open` throughout transit (updating only `current_position`) skipped the check entirely тАФ `wait_for_target` was cleared immediately, and the next position update was misidentified as a user move. The fix applies the direction/progress check based on position delta regardless of the HA state string. The step-motor mid-transit pause (Issue #186) is checked first so it still takes priority, and `stopped` state is excluded since it's an unambiguous halt signal.

- **Diagnostic event buffer label and transit timeout placement (#298):** The "Manual override event buffer size" label on the Debug & Diagnostics screen is renamed to "Diagnostic event buffer size" тАФ the ring buffer is shared across manual override, motion, weather, time-window, cover commands, and the pipeline, not just manual override. The Transit timeout setting is moved from the Debug screen to the Manual Override screen, where it belongs (it controls how long to wait after a cover stops moving before treating the next position report as a user-initiated override). The config summary now shows transit timeout in the manual override section.

- **Geometry menu button label mismatch (#289):** The options-flow menu button for the geometry step was labeled "Window Geometry" while the step title read "Cover Geometry". The label is now "Cover Geometry" across all three languages (en/de/fr).
