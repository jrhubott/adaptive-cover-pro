## 🐛 Bug Fixes

### Fix glare zone — zones now take effect when climate mode is active (Refs #231)
When climate mode was enabled, the climate handler was computing a solar-tracking position directly for the GLARE_CONTROL strategy, outprioritizing the glare zone handler and making glare zones completely ineffective. The fix changes ClimateHandler to return `None` for GLARE_CONTROL, allowing the glare zone handler (priority 45) to fire naturally below climate (priority 50). When no glare zones are active the pipeline falls through to the solar handler as before.

**Note:** This is a deliberate behaviour change. Users with both climate mode and glare zones configured will now see glare zone positions take effect when a zone is active. If this causes unexpected interactions please report them on issue #231.
