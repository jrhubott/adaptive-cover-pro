## 🐛 Fix Legacy Adaptive Cover Import Detection

### Bug Fixes

- **Fix import from Adaptive Cover not appearing** — The import menu option was never shown because the detection required the old integration's config entries to be in a `LOADED` state. When Adaptive Cover Pro replaces the original Adaptive Cover integration, the old entries remain in `NOT_LOADED` or `SETUP_ERROR` state (no code left to load them), so the menu was always hidden. The detection now finds legacy entries regardless of their load state.

- **Fix multi-entry import silently dropping entries** — The import selection was previously a multi-select, but the HA config flow terminates after creating one entry. Only the first selected entry was ever imported; all others were silently dropped. Changed to single-select with a note to re-run the flow for additional entries.

### Testing

- Added 10 unit tests for import detection across all entry states
- 368 tests passing

### Installation

Download and install via HACS or manually copy `custom_components/adaptive_cover_pro` to your HA config directory.
