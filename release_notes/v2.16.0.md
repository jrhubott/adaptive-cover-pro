## 🎯 Manual Override Persistence & Smarter Settings Copy

### ✨ Features

- **Persist manual override across restarts** (#192, PR #195) — Manual overrides now survive Home Assistant reboots and integration reloads. The per-entity expiry map is restored on startup; covers with an active manual override skip automatic repositioning on first refresh.

- **Split "Copy to Other Covers" categories into values vs. sensors** (#125, PR #196) — Copying settings no longer forces you to re-assign room-specific sensors (indoor temperature, presence, motion, custom position triggers, etc.) just to push a global threshold change. Each mixed category (Light & Cloud, Climate Mode, Motion Override, Force Override, Custom Positions, Weather Override) is now split into a `— Thresholds` sub-category (safe to copy globally) and a `— Sensors` sub-category (room-specific entity assignments). Both can still be selected together for a full copy. Legacy category keys are preserved for backward compatibility.

### 🐛 Bug Fixes

- **Reset button now respects time window and auto-control gates** (#193, PR #194) — The reset button previously sent a cover move even when outside the active time window or when Automatic Control was OFF. It now follows the same gates as all other automated moves.
