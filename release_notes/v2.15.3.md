## 🐛 Bug Fix: Covers no longer move when Automatic Control is disabled

### 🐛 Bug Fixes
- **Covers no longer reposition at end of time window when Automatic Control is OFF** — When the operational time window closed and *Return to Default* (`return_sunset`) was enabled, covers would move to the effective default position regardless of the Automatic Control toggle. Root cause: the end-of-window command used `force=True` which bypassed the auto-control gate. Fixed by adding the same early-return guard used by the manual-override-expiry path.

### 🧪 Testing
- Added regression test: `test_window_close_skips_reposition_when_auto_control_off`
- Added happy-path test: `test_window_close_sends_reposition_when_auto_control_on`
