# v2.13.2 — Motion Override Fixes

## 🐛 Bug Fixes

- **Motion Status sensor initialization**: Fixed sensor showing `waiting_for_data` indefinitely when all motion sensors were already off at startup. The feature now initializes to `no_motion` immediately on first refresh.
- **Motion timeout position calculation**: Fixed state property bypassing the pipeline result, which was skipping min/max position limits, interpolation, and inverse state transforms. Motion timeout positions now flow through the full pipeline for correct limit enforcement.
- **Motion sensor entity selector**: Added `presence` device class to the motion sensor entity selector so Bluetooth presence sensors (ESPresense, room presence, etc.) can be selected alongside motion and occupancy sensors.

## 🧪 Testing

- Updated position calculation test to verify pipeline result is used instead of raw default position
- Added 5 new tests for startup initialization and Motion Status sensor display logic
- All 1017+ existing tests pass

## 📋 Compatibility

- Home Assistant 2024.5.0+
- Python 3.12+
